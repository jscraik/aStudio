# Documentation QA Checklist

Last updated: 2026-01-04

## Doc requirements
- Audience: Documentation maintainers and reviewers
- Scope: QA checklist for Markdown documentation updates
- Non-scope: Code review checklists or CI enforcement rules
- Owner: TBD (confirm)
- Review cadence: TBD (confirm)

## Checklist

- [ ] Purpose and audience are explicitly stated.
- [ ] Scope and non-scope are explicit and non-overlapping.
- [ ] Doc owner and review cadence are defined.
- [ ] Headings are informative sentences and follow a logical hierarchy.
- [ ] Table of contents is present for long documents.
- [ ] Paragraphs are short, skimmable, and lead with topic sentences.
- [ ] Links are descriptive and resolve correctly.
- [ ] Code blocks match actual commands and files.
- [ ] Security and privacy guidance is correct and not overly permissive.
- [ ] Accessibility basics are met (no color-only cues, alt text for images).
- [ ] Risks and assumptions are documented for operational steps.
- [ ] Verify steps and troubleshooting are included when procedures exist.
- [ ] Last updated date is present and current.
- [ ] Terminology and naming are consistent with the codebase.
