# Code Documentation QA Checklist

Last updated: 2026-01-04

## Doc requirements
- Audience: Documentation maintainers and reviewers
- Scope: QA checklist for code-level docs (JSDoc/DocC/config docs)
- Non-scope: User-facing Markdown guides and policies
- Owner: TBD (confirm)
- Review cadence: TBD (confirm)

## Checklist

- [ ] Public APIs are documented with clear summaries.
- [ ] Parameters, returns, and error conditions are documented.
- [ ] Examples reflect actual usage and compile or run.
- [ ] Accessibility contracts are documented for UI components.
- [ ] Threading or concurrency expectations are explicit.
- [ ] Security-sensitive parameters are highlighted.
- [ ] Deprecated APIs include migration guidance.
- [ ] Comments explain constraints and behavior, not restating code.
